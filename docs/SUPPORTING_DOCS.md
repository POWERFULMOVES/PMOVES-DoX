# Supporting Documentation & Auto-Tagging

This document outlines the strategy for creating documentation that is **auto-tagged based on source** and **cross-referenced with the API**.

## 1. Auto-Tagging Schema

All documentation files should include YAML frontmatter to support auto-tagging by the ingestion pipeline.

```yaml
---
title: Feature Name
source: [api-spec | manual | code-analysis | user-guide]
status: [active | planned | deprecated]
api_refs:
  - GET /path/to/endpoint
  - POST /path/to/endpoint
tags: [finance, data-set, security]
---
```

### Source Types
- `api-spec`: Generated from OpenAPI/Swagger definitions.
- `manual`: Written by humans (e.g., guidelines, plans).
- `code-analysis`: Generated by analyzing the codebase (e.g., Docling).

- **Governance:** Review and approval workflows for auto-generated tags.

## 3. API Cross-Referencing

Documentation should explicitly link to API definitions.

**Example:**
> To trigger the financial analysis, use the [Analyze Financials API](docs/API_REFERENCE.md#get-analysisfinancials).

**Auto-Linking:**
The documentation pipeline will scan for `api_refs` in the frontmatter and automatically inject "Related API" links at the bottom of the rendered page.
